# Huffman 文件压缩工具

基于霍夫曼编码的无损文件压缩工具，支持单文件、多文件、通配符匹配和目录递归压缩。

## 功能特性

- 单文件压缩与解压
- 多文件归档压缩
- 通配符支持（* 和 ?）
- 目录递归压缩与解压
- 无损压缩，保证数据完整性
- CRC32 校验

## 编译

```bash
make -f makefile.win
```

或使用 Dev-C++ 打开 `Huffman.dev` 项目文件。

## 使用方法

### 帮助信息
```bash
./Huffman.exe h
```

### 单文件压缩/解压
```bash
# 压缩
./Huffman.exe c <输入文件> <输出文件>

# 解压
./Huffman.exe d <输入文件> <输出文件>
```

### 多文件压缩/解压
```bash
# 压缩多个文件
./Huffman.exe a <归档文件> <文件1> <文件2> ...

# 列出归档内容
./Huffman.exe l <归档文件>

# 解压归档
./Huffman.exe x <归档文件> <输出目录>
```

### 通配符压缩
```bash
# 使用 * 通配符
./Huffman.exe a archive.huff *.txt

# 使用 ? 通配符
./Huffman.exe a archive.huff file?.txt
```

### 目录递归压缩
```bash
# 压缩目录
./Huffman.exe r <目录> <归档文件>

# 解压
./Huffman.exe x <归档文件> <输出目录>
```

### 自动测试
```bash
# 单文件测试
./Huffman.exe t <文件>

# 多文件测试
./Huffman.exe m
```

## 算法原理

### 霍夫曼编码

霍夫曼编码是一种贪心算法，通过构建最优前缀编码树来实现数据压缩：

1. **统计频率**：统计文件中每个字节的出现频率
2. **构建最小堆**：将每个字节作为叶子节点，频率作为权重
3. **构建霍夫曼树**：每次取出两个最小权重的节点合并，直到只剩一个根节点
4. **生成编码**：从根节点到叶子节点的路径即为该字节的编码（左0右1）
5. **编码数据**：用生成的变长编码替换原始字节数据

### 压缩流程

1. 读取原始文件，统计字节频率
2. 构建霍夫曼树，生成编码表
3. 将霍夫曼树信息写入压缩文件头
4. 使用编码表对数据进行编码并写入
5. 计算并写入 CRC32 校验值

### 解压流程

1. 读取压缩文件头，获取霍夫曼树信息
2. 重建霍夫曼树
3. 根据霍夫曼树解码压缩数据
4. 验证 CRC32 校验值
5. 写入解压后的文件

## 关键修复

### Bug 1：多文件压缩时归档损坏

**问题**：header 覆盖压缩数据导致归档文件损坏

**解决**：使用临时文件，先写压缩数据到临时文件，再按顺序写入 header 和数据

### Bug 2：多文件解压时文件指针定位不准

**问题**：解压后文件指针未正确移动到下一个文件位置

**解决**：记录数据起始位置，解压后使用 fseek 精确定位到下一个文件
